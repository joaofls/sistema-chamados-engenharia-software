# Casos de Uso

## Atores

- Solicitante
- Analista de Suporte
- Administrador

## Casos de uso principais

### Abrir chamado

O solicitante informa título, descrição e prioridade. O sistema cria o ticket com status inicial `aberto`.

### Consultar chamado

Solicitante, analista e administrador podem visualizar dados do ticket, incluindo comentários e histórico de status.

### Comentar chamado

Usuários podem adicionar interações em tickets não fechados para complementar o atendimento.

### Atribuir responsável

O chamado pode ser direcionado para um analista de suporte ou administrador responsável pelo atendimento.

### Alterar status

O status do chamado pode evoluir durante o atendimento e cada mudança é registrada no histórico.

### Gerenciar usuários

O sistema permite cadastro e consulta de usuários internos.

### Visualizar relatórios

O administrador pode consultar relatórios agregados por status, prioridade e responsável.
